# StrangePlaces项目 Agent 规则（强制遵守）

## 语言规则（强制）

- 本项目内：所有规则文档、技能文档、以及 Agent 的所有回答，一律使用中文。
- 如必须保留英文术语（例如类名/接口名/Unity 组件名/路径/命令）可保留英文，但解释文本必须为中文。

## Unity 场景修改（强制）

- 任何对 Unity 场景/层级/Prefab 的改动（创建/移动/删除 GameObject、修改组件字段、摆放关卡物体、修改 UI 文本、保存场景等）必须通过 Unity MCP（mcp-unity）完成。
- 禁止手工编辑 `.unity` / `.prefab` / `.asset` 的 YAML 来改场景内容（除非用户明确要求并确认 MCP 不可用）。
- 每次通过 MCP 改完：
  - 调用保存（例如 `save_scene`）
  - 拉取并处理 Console error（例如 `get_console_logs(logType="error")`）

## 游戏文本规则（强制）

- 所有游戏内可见文本（UI、提示牌、结算等）必须为中文。

## 编码规则（强制）

- 全项目脚本文件（例如 `.cs`）必须使用 UTF-8 编码（推荐 UTF-8 无 BOM）。
- 禁止使用 ANSI/GBK 等本地代码页保存脚本，避免出现“中文乱码/编译失败/不可逆损坏”。

## 行尾格式（强制）

- 所有脚本文件（例如 `.cs`）行尾必须为 CRLF。

## PowerShell 行尾修复禁令（强制）

- 禁止使用 PowerShell 把真实换行替换成字面量 `\\r\\n`（两个字符序列），例如：`-replace "\\\\r?\\\\n", "\\\\r\\\\n"`。
  - 这会把文件变成“单行超长文本”，并导致脚本/中文注释看起来乱码、编译报错且难以恢复。
- 统一换行只能写入真实 CRLF：使用反引号转义 `` `r`n `` 或其它能写入“实际换行”的方式进行写入。
